# deribit-price-service

Приложение предоставляет внешнее API на FastAPI для получения:
- всех сохранённых цен по указанному тикеру
- последней цены
- цен за указанный диапазон времени

---

# Как запустить проект

```bash
1. git clone https://gitlab.com/Loopeq/deribit-price-service.git

2. mv .env.example .env
На этом моменте в .env файле нужно указать произвольный POSTGRES_PASSWORD

3. docker compose up --build

4. http://localhost:8000/docs

```

# API
Допуступные тикеры - `btc_usd` и `eth_usd` 
- GET /price/?ticker=btc_usd - получить все сохраненные цены по тикеру

- GET /price/latest?ticker=btc_usd - получить самую молодую цену по тикеру

- GET /price/range?ticker=btc_usd&from_ts=1700000000&to_ts=1700003600 - получить список цен в диапазоне UNIX дат

# Design decisions

Приложение разделено на слои:

- `client` — взаимодействие с API Deribit  
- `service` — бизнес-логика получения цен  
- `repository` — репозиторий для получения данных из базы  
- `api` — HTTP интерфейс  

Это позволяет уменьшить связанность компонентов и упростить тестирование.

---

**Слой репозитория**

Работа с базой данных вынесена в `PriceRepository` для создания прослойки между бизнес-логикой приложения и источником данных (БД, API). Он позволяет заменить прямой доступ к базе данных на работу с объектами через интерфейс, обеспечивая независимость кода, удобство тестирования. 

---

**Сервисный слой**

Основная бизнес-логика реализована в `PriceCollectorService`, а celery выполняет только запуск сервиса. Позволяет групировать и переиспользовать бизнес-логику и зависимости для её реализации. 

---

**Клиент**

В клиент вынесена логика работы со сторонним API для удобства тестирования, и для изоляции от внешнего кода
